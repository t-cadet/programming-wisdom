Adding Sparse Unicode Support

Today we're tackling the complex world of Unicode. Instead of supporting the entire specification, we'll focus on a "sparse" implementation that covers the most common use cases for game development text rendering.

The Problem with Unicode

Full Unicode support is a massive undertaking, involving complex rules for combining characters, bidirectionality, and more. For many games, this is overkill. We only need to render text from a limited set of languages.

A Simpler Approach

Our approach will be to pre-process our font assets to contain only the glyphs we need. We'll map UTF-8 input to our internal glyph indices. This keeps the runtime code simple and fast.
