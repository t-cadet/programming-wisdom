Why No Frameworks?

This blog itself is an experiment in minimalist single-page application design. By avoiding large frameworks, we gain complete control, reduce dependencies, and achieve maximum performance for this specific use case.

Core Principles

  • Single File: HTML, CSS, and JS are all in `index.html`. This simplifies deployment to any static host.
  • Static Content: Articles are separate HTML files, not database entries. This makes them easy to edit and version control.
  • Vanilla JS: No React, Vue, or Angular. We manipulate the DOM directly, which is perfectly fine for an application with simple state transitions.
  • State Management: A single, simple JavaScript object holds the entire application state (the list of articles, sort order, etc.). Rendering functions are called when the state changes.

// The "framework" is just a few functions:
function render() {
    // Update DOM based on `state` object
}

    // Mutate `state` object
    // Call render()
