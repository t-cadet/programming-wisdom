Building a syntax highlighter from scratch seems daunting, but a simple, regex-based tokenizer for a language like C is surprisingly manageable. This article explores the approach used in this very blog.

The Strategy

The core idea is to consume the input string piece by piece, always looking for the longest possible match from a set of predefined token types. We prioritize tokens to handle ambiguities (e.g., checking for a multi-line comment before a single `/` character).

The order of matching is crucial:

  1. Preprocessor directives (entire lines starting with #)
  2. Multi-line comments (/* ... */)
  3. Single-line comments (// ...)
  4. Strings and character literals
  5. Numbers (including hex and scientific notation)
  6. Keywords and Identifiers

Example Code

Here is the tokenizer in action on a sample C file. Notice how keywords, comments, strings, and numbers are colored, but identifiers like printf or my_variable are not.


#include 

// This is a single-line comment.
#define PI 3.14159

/*
 * This is a multi-line comment
 * that spans multiple lines.
 */
int main() {
    char greeting[] = "Hello, World!\n";
    int hex_value = 0x1A;
    float e_notation = 1.23e-4;
    
    // A simple loop
    for (int i = 0; i < 5; i++) {
        printf("%s", greeting); // Function calls are not keywords
    }

    return 0; // Return success
}
