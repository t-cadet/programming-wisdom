Providing a C Function for Every Linux Syscall

Previously: ⬅ Wrapping Linux Syscalls in C.

I've been writing a single file, header-only library for Linux's API. Previously, I gathered all of Linux's syscall numbers in a table, and wrote generic SyscallN wrappers to use them.

Still, a dedicated C function would be more convenient to use than a generic wrapper, so let's provide one for each syscall.

Full source here.

I. Gathering syscall prototypes

The first step is to retrieve all of the prototypes.

linux/Documentation/process/adding-syscalls.rst

The new entry point also needs a corresponding function prototype, in
``include/linux/syscalls.h``, marked as asmlinkage to match the way that system
calls are invoked:

    asmlinkage long sys_xyzzy(...);

Let's take a look at that syscalls.h file.

linux/include/linux/syscalls.h

asmlinkage long sys_io_setup(unsigned nr_reqs, aio_context_t __user *ctx);
asmlinkage long sys_io_destroy(aio_context_t ctx);
asmlinkage long sys_io_submit(aio_context_t, long, 
			struct iocb __user * __user *);
asmlinkage long sys_io_cancel(aio_context_t ctx_id, struct iocb __user *iocb, 
			      struct io_event __user *result);

That seems promising.

So I went ahead and used a script to parse all of the prototypes from the syscalls.h file, retrieving the name of the syscalls, as well as their parameters as a list of name and type pairs. However, I do not particularly recommend that approach. Using one of the more thorough third-party syscall tables, like this one based on kernel images built with debug info, is probably easier (I only found out about it after the fact).

Here are some of the issues I had with syscalls.h:

  • some prototypes are missing, for example mmap
  • some syscalls appear multiple times with different prototypes due to #ifdefs, for example clone
  • some parameters only have types, no name, for example asmlinkage long sys_listen(int, int)
  • pointer types are often specified as unsigned long instead of void *, for example in asmlinkage long sys_munmap(unsigned long addr, size_t len)
  • const is often missing from read-only pointers, for example in asmlinkage long sys_sendmsg(int fd, struct user_msghdr __user *msg, unsigned flags)
  • parameters have clearer names in the manual

All in all, I ended up adding or overriding 197 prototypes, which is approximately 40% of the 485 syscalls listed in my syscall numbers table. I then used the prototype info to generate what I call "trivial wrappers" (as opposed to wrappers that require manual override) for every syscall.

trivial wrappers for read and write

// 5b. Reading and writing file data
long read_linux(unsigned int fd, void *buf, unsigned long count) {
  return Syscall3_linux(NR_read_linux, fd, buf, count, 0);
}
long write_linux(unsigned int fd, const void *buf, unsigned long count) {
  return Syscall3_linux(NR_write_linux, fd, buf, count, 0);
}

II. Retrieving Linux headers' constants and types

Most Linux syscalls cannot be used conveniently on their own; they require constants.

seek positions

#define SEEK_SET_linux              0
#define SEEK_CUR_linux              1
#define SEEK_END_linux              2

And they require types.

clone arguments

typedef struct {
  unsigned long long flags;
  unsigned long long pidfd;
  unsigned long long child_tid;
  unsigned long long parent_tid;
  unsigned long long exit_signal;
  unsigned long long stack;
  unsigned long long stack_size;
  unsigned long long tls;
  unsigned long long set_tid;
  unsigned long long set_tid_size;
  unsigned long long cgroup;
} clone_args_linux;

Extracting all of these from Linux headers manually is a lot of work.

To make the process faster, I plugged Gemini CLI to a folder containing the Linux sources and the manual sources. Then going through the subsections of my syscall numbers table one at a time, I asked the LLM to retrieve all relevant constants and types for the subsection. I then checked the results trying to make sure it didn't forget or mess up anything, and moved to the next subsection.

It still took a very long time to go through all subsections, as you might guess when looking at the thousands of lines of defines and structs in the end result. Due to the nature of the process, there may be missing symbols or other issues that will get fixed as I use the library more.

III. Syscalls that require non-trivial wrappers

Some syscall wrappers do not "just work" like the ones for read and write I showed above, and require a manual implementation. There can be a few reasons for that. Let's go through them:

In the above table, you can see that some syscalls do not have a syscall number on every architecture.

For example, such a pedestrian syscall as fork is not implemented on arm64 nor on riscv64. Linux leaves it up to user space to figure out what fork does and how to fill the gap in the API. Thus, the man and your favorite flavor of libc are your friends for this part of the process.

fork wrapper

long fork_linux(void) {
  return clone_linux(SIGCHLD_linux, 0, 0, 0, 0);
}

Usually the implementation is not too bad, but it can get a lot more involved.

wait4 wrapper

long wait4_linux(int pid, int *stat_addr, int options, rusage_linux *ru) {
#if !(defined(__riscv) && (__riscv_xlen == 32))
  return Syscall4_linux(NR_wait4_linux, pid, stat_addr, options, ru, 0);
#else
  int which = P_PID_linux;
  if (pid < -1) {
    which = P_PGID_linux;
    pid = -pid;
  } else if (pid == -1) {
    which = P_ALL_linux;
  } else if (pid == 0) {
    which = P_PGID_linux;
  }

  siginfo_t_linux infop;
  infop.si_pid_linux = 0;

  long ret = Syscall5_linux(NR_waitid_linux, which, pid, &infop, options | WEXITED_linux, ru, 0);

  if (ret >= 0) {
    ret = infop.si_pid_linux;
    if (infop.si_pid_linux && stat_addr) {
      switch (infop.si_code) {
        case CLD_EXITED_linux: *stat_addr = (infop.si_status_linux & 0xff) << 8; break;
        case CLD_KILLED_linux: *stat_addr = infop.si_status_linux & 0x7f; break;
        case CLD_DUMPED_linux: *stat_addr = (infop.si_status_linux & 0x7f) | 0x80; break;
        case CLD_TRAPPED_linux:
        case CLD_STOPPED_linux: *stat_addr = (infop.si_status_linux << 8) | 0x7f; break;
        case CLD_CONTINUED_linux: *stat_addr = 0xffff; break;
        default: *stat_addr = 0; break;
      }
    }
  }

  return ret;
#endif
}

gethostname wrapper

long gethostname_linux(char *name, unsigned long len) {
  if (name) {
    utsname_linux uts;
    long res = uname_linux(&uts);
    if (res < 0) return res;
    long i = 0;
    while (i < len && uts.nodename[i]) {
      name[i] = uts.nodename[i];
      ++i;
    }
    if (i < len) {
      name[i] = '\0';
      return 0;
    } else if (len > 0) {
      name[len - 1] = '\0';
    }
  }
  return -ENAMETOOLONG_linux;
}

B. Different architectures have different prototypes for the same syscall

Usually it's because passing 64-bit arguments on 32-bit architectures requires splitting the argument in LO and HI parts to make it fit into 32-bit registers, sometimes with the additional requirement that the 64-bit argument must be passed starting on an even register, for example 0-1, 2-3, or 4-5, but not 1-2. This can require padding.

pread64 wrapper

long pread64_linux(unsigned int fd, void *buf, unsigned long count, long long pos) {
#if defined(__x86_64__) || defined(__aarch64__) || (defined(__riscv) && (__riscv_xlen == 64))
  return Syscall4_linux(NR_pread64_linux, fd, buf, count, pos, 0);
#elif defined(__i386__)
  return Syscall5_linux(NR_pread64_linux, fd, buf, count, LO32_bits(pos), HI32_bits(pos), 0);
#elif defined(__arm__) || (defined(__riscv) && (__riscv_xlen == 32))
  return Syscall6_linux(NR_pread64_linux, fd, buf, count, 0, LO32_bits(pos), HI32_bits(pos), 0);
#endif
}

Other times arguments are just swapped for historical reasons.

clone wrapper

long clone_linux(unsigned long flags, void *stack, int *parent_tid, int *child_tid, unsigned long tls) {
#if defined(__x86_64__)
  return Syscall5_linux(NR_clone_linux, flags, stack,  parent_tid, child_tid, tls, 0);
#else
  return Syscall5_linux(NR_clone_linux, flags, stack,  parent_tid, tls, child_tid, 0);
#endif
}

C. 32-bit to 64-bit transition

Syscalls dealing with time or file sizes had register sized arguments, which are not adequate to handle times after 2038 or files bigger than 4GiB on 32-bit architectures.

Indeed, on Linux, time is represented as a signed 32-bit integer counting the number of seconds elapsed since the Unix epoch (00:00:00 UTC on January 1, 1970). At 03:14:07 UTC on January 19, 2038, that counter will reach the int's maximum value, 2147483647, and wrap around to -2147483648, which represents December 13, 1901 (yes, it was a Friday).

As for files, the exact limit is 4GiB - 1, because 0 is a valid value, or even 2GiB - 1 when a signed 32-bit integer is used to represent the size. How much of an issue is it? Well, video files and game assets can get quite big quickly: a single hour of HD video can exceed 4GiB.

So Linux introduced new syscalls to deal with the issue.

clock_gettime syscall numbers

/*║                         Syscall Name                         ║ x86_64  │  arm64  │ riscv64 │ x86_32  │  arm32  │   riscv32   ║*/
/*╟──────────────────────────────────────────────────────────────╨─────────┴─────────┴─────────┴─────────┴─────────┴─────────────╢*/
/*║*/ #define NR_clock_gettime_linux                BY_ARCH_linux(      228,      113,      113,      265,      263,     void) /*║*/
/*║*/ #define NR_clock_gettime64_linux              BY_ARCH_linux(     void,      403,     void,      403,      403,      403) /*║*/

Except the new syscalls are not implemented on 64-bit architectures. Why, of course! The original implementation already uses 64-bit arguments on 64-bit architectures. No need to provide a new syscall.

So to get a consistent API, user-space has to create a custom wrapper for all of these syscalls.

clock_gettime64 wrapper

// Disabled wrapper: long clock_gettime_linux(int which_clock, __kernel_old_timespec_linux *tp);
long clock_gettime64_linux(int which_clock, __kernel_timespec_linux *tp) {
#if defined(__x86_64__) || defined(__aarch64__) || (defined(__riscv) && (__riscv_xlen == 64))
  return Syscall2_linux(NR_clock_gettime_linux, which_clock, tp, 0);
#else
  return Syscall2_linux(NR_clock_gettime64_linux, which_clock, tp, 0);
#endif
}

Out of these my favorite has to be fadvise64_64. It looks like they foresaw the need for a 64-bit offset and created fadvise64. But then they also needed a 64-bit length which resulted in fadvise64_64.

F. The case of vfork

vfork is a bold system call.

If you let it return from a function (such as a vfork wrapper function), it will corrupt the stack.

The idea is that vforks lets you fork processes efficiently, by sharing memory between the parent and child processes, including the stack. The downside of that approach is that when the child process returns from a function, it pops the stack frame. So then when the parent process is unblocked, and tries to return from the same function with an already popped stack frame, bad things happen.

So I disabled vfork. The modern way to do efficient forks is to use clone3 with appropriate arguments. But even clone3 does not allow you to create a function wrapper that shares the stack, unless you use assembly to handle the stack frame yourself. I did not implement that since using clone3 to share all of the memory except the stack gets you most of the benefits of vfork with none of the issues.

I only gave a few examples, but I had to disable 121 wrappers, or about a quarter of the 485 total syscalls, and provide non-trivial wrappers for another 117 syscalls.

Looking at the above wrappers makes it quite clear that there is value in having a dedicated function for each syscall. Dealing with these API esoterica everywhere syscalls are used would be a mess.

IV. Design Choices

I try to stick close to the Linux API, to provide access as "raw" as possible to Linux.

I nevertheless made a few changes:

  • I removed trivial typedefs, so for example pid_t is just int
  • I infer size arguments when possible, for example for signal masks
long rt_sigprocmask_linux(int how, unsigned long long *set, unsigned long long *oset) {
  return Syscall4_linux(NR_rt_sigprocmask_linux, how, set, oset, sizeof(*set), 0);
}
  • I keep 64-bit variants of syscalls: fcntl64_linux is defined and fcntl_linux is disabled

Note that Linux syscalls cover most but not all of the user-space Linux API. For example there is also the ELF runtime (command-line arguments, environment variables, auxiliary vector), the vDSO, and special filesystems like /proc and /sys.
